Which is which? A quick orientation

All three solve “let me use this thing without re-entering credentials everywhere,” but they were born for different worlds.

 OAuth 2.0OpenID ConnectSAML 2.0
Mainly doesAuthorization (delegated access)Authentication (login) on top of OAuthAuthentication + SSO (and attributes)
Data formatTokens (often JWT)ID token (JWT)XML assertions

// rule of thumb

Building a new app for the open web or mobile? Reach for OAuth 2.0 + OpenID Connect. Integrating with an enterprise that already runs central SSO? You’ll likely meet SAML 2.0. Plenty of organisations run both.
