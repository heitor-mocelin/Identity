// OAuth 2.0 & OpenID Connect, explained from scratch

The keys to your data, without giving away your password.

Every time you click “Sign in with…” or “Allow this app to access your…”, two quiet protocols do the heavy lifting. This guide explains why they exist, how they work, and what each moving part does — starting with everyday analogies and gradually going as deep as the specs themselves.

The 30-second version

OAuth 2.0 is about authorization — letting one app do something on your behalf in another app (e.g. “let this photo printer read my photos”) without handing over your password. OpenID Connect is a thin layer on top of OAuth that adds authentication — proving who you are, so an app can log you in.

OAuth 2.0 = “what can this app do?” OpenID Connect = “who is this user?”

A picture is worth a thousand redirects

The canonical OAuth 2.0 dance, straight from RFC 6749 — four players passing a request around:

abstract-protocol-flow — RFC 6749 §1.2
     +--------+                               +---------------+
     |        |--(A)- Authorization Request ->|   Resource    |
     |        |                               |     Owner     |
     |        |<-(B)-- Authorization Grant ---|               |
     |        |                               +---------------+
     |        |
     |        |                               +---------------+
     |        |--(C)-- Authorization Grant -->| Authorization |
     | Client |                               |     Server    |
     |        |<-(D)----- Access Token -------|               |
     |        |                               +---------------+
     |        |
     |        |                               +---------------+
     |        |--(E)----- Access Token ------>|    Resource   |
     |        |                               |     Server    |
     |        |<-(F)--- Protected Resource ---|               |
     +--------+                               +---------------+

Don’t worry if this looks cryptic right now — by the end of the Roles and Flow pages it will read like plain English.
